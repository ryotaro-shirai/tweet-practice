## フォローリクエストをする

**アクター:** ログイン済みユーザー 
**事前条件:** 対象ユーザーがプライベートアカウントである/まだフォローしていない
**事後条件:** フォローリクエストが「申請中」状態で記録されている

**基本フロー:**
1. ユーザーが対象ユーザーのプロフィールページを表示する
2. システムが「フォローリクエストを送る」ボタンを表示する
3. ユーザーがボタンを押す
4. システムがフォローリクエストを「申請中」状態で記録する 
5. システムが対象ユーザーに通知する

**代替フロー:**
- 対象ユーザーがパブリックアカウントの場合: 即時フォロー成立（別ユースケース）

**例外フロー:**
- すでに申請中のリクエストが存在する場合: エラーを返す


## フォローリクエストを承認する
**アクター:** ログイン済みユーザー 
**事前条件:** 対象ユーザーにフォローリクエストが届いている
**事後条件:** フォローリクエストが「承認」状態で記録されている

**基本フロー:**
1. ユーザーが承認リクエストページを表示する
2. システムが「承認」ボタンと「拒否」ボタンを表示する
3. ユーザーが「承認」ボタンを押す
4. システムがフォローリクエストを「承認」状態、フォローリクエストを送ったユーザーがユーザーをフォローしたことを記録する
5. システムがフォローリクエストを送ったユーザーにフォローが承認されたことを通知する

**代替フロー:**
- 特になし

**例外フロー:**
- 特になし

## 単純リポストを投稿する
**アクター:** ログイン済みユーザー 
**事前条件:** 特になし
**事後条件:** 単純リポストが記録されていること

**基本フロー:**
1. ユーザーが単純リポストしたいツイートを表示する（自分自身のツイートも可能。タイムラインでもツイート詳細でも可能）
2. システムが「単純リポスト」ボタンを表示する
3. ユーザーが「単純リポスト」ボタンを押す
4. システムが単純リポストとして記録する。

**代替フロー:**
- 特になし

**例外フロー:**
-  特になし

## 引用ツイートを投稿する
**アクター:** ログイン済みユーザー 
**事前条件:** 特になし
**事後条件:** 引用ツイートがツイートとして記録されていること

**基本フロー:**
1. ユーザーが引用ツイートしたいツイートを表示する（タイムラインでもツイート詳細でも可能）
2. システムが「引用ツイート」ボタンを表示する
3. ユーザーが「引用ツイート」ボタンを押す
4. システムが投稿欄、「投稿」ボタンを表示する
5. ユーザーがコメントを入力し、投稿ボタンを押す
6. システムがコメントの内容をツイートとして記録する。その時にその投稿に紐づけて引用ツイート対象のツイートのidも引用ツイート元のツイートとして記録する

**代替フロー:**
- 特になし

**例外フロー:**
- 文字が空である場合、文字が140文字を超える場合など通常の投稿フローと同じバリデーションを行う。バリデーション違反の場合にはメッセージを出して、修正を促す。


## タイムライン取得フロー（ログインユーザー）
**アクター:** ログインユーザー
**事前条件:** 特になし
**事後条件:** タイムラインが表示されていること

**基本フロー:**
1. ユーザーがタイムラインを開く
2. システムがユーザーがフォローしているユーザーのツイートとリポストを検索し、投稿日時の降順に表示する（100件）。また最下部には「追加表示」ボタンを表示する。
3. ユーザーが最下部までスクロールし、「追加表示」ボタンを押す
4. 再度システムは追加の100件を取得し、投稿日時の降順に追加表示する（100件）。また最下部には「追加表示」ボタンを表示する。

**代替フロー:**
- 特になし

**例外フロー:**
- ユーザーがフォローしているユーザーの全体の投稿数が足りなくなった場合にはそこで打ち止めとする。

3. ユーザーが最下部までスクロールし、「追加表示」ボタンを押す
4. 再度システムは追加の100件を取得し、投稿日時の降順に追加表示する（100件）。また最下部には「追加表示」ボタンを表示する。

**代替フロー:**
- 特になし

**例外フロー:**
- 特になし
